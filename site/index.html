Why this exists

You re-read your notes more than you read them.
Listen instead.

Most note-takers spend more time re-reading their own notes than writing them. Walking, commuting, cooking — that time is reading time you're leaving on the floor. Bedrock Voice turns any note into spoken audio you can take with you, with karaoke captions that follow each word when you do come back to the screen.

It's also the canonical reading accommodation for ADHD, dyslexia, and auditory processing. Bookshare, Kurzweil 3000, and a generation of school-library reading tools are built on the same bi-modal pattern (seeing + hearing simultaneously). Those are SaaS and locked-in. This is opinionated for Obsidian, local-first, MIT.

What it does

Four moves. One outcome.

01

Reads any note aloud

Clean local voice via Kokoro 82M (Apache-2). Apple-Silicon native, CPU is enough.

02

Karaoke captions

A floating HUD highlights the current word and current sentence as they're spoken. The next sentence previews underneath.

03

Tightened for cadence

A small LLM pass first compresses the note into a spoken script — using auth you already have. Your note as cadence, not text-dump.

04

Smart cache

Click twice on an unchanged note → instant replay. Edit the note → next click regenerates. No compute waste.

For anyone whose brain doesn't read in a straight line.

How it works

A thin plugin. A small pipeline. A cached audio file.

Obsidian note ──▶ Plugin (TS) ──▶ Python pipeline ──▶ Karaoke HUD
                                  │
                                  ├─ Extract source (tldr + H2, or whole body)
                                  ├─ Tighten for speech (Claude/Codex CLI, or local MLX)
                                  ├─ Synthesize sentence-by-sentence (Kokoro)
                                  └─ Return wav + per-sentence timing JSON

The plugin is a thin shell. All heavy work happens in Python and gets cached on disk. No recompute when nothing changed. Full architecture notes: docs/architecture.md.

Why it stays on your machine

No notes leave. No API key required. No meter ticks.

Component Where it runs What it costs
TTS (Kokoro 82M) Locally, on your CPU $0
Default backend — script tightening (local) MLX-LM in-process $0 — fully offline
Optional — script tightening (claude) Claude Code CLI auth $0 marginal · if you already pay
Optional — script tightening (codex) OpenAI Codex CLI auth $0 marginal · if you already pay

The honest version: the default path is $0 with no external auth required. The CLI backends are faster but only free if you already have those subscriptions — we made local the default so the no-meter promise holds for everyone.

Install

Three steps, ten minutes.

v0.1 requirement: macOS Apple Silicon. The installer fails fast with a clear message on other platforms. Windows, Linux, and Intel Macs are on the roadmap.

  1. 1. Plugin via BRAT

    Install BRAT from Obsidian community plugins, then BRAT → "Add Beta Plugin" → paste:

    opendian/bedrock

    Enable Bedrock Voice under Community plugins.

  2. 2. One command for the pipeline

    Open Terminal and run:

    cd "<your-vault>/.obsidian/plugins/bedrock-voice/voice" && bash install.sh

    The installer verifies platform, installs Homebrew (if missing) + Python 3.12 + Kokoro + MLX-LM, smoke-tests the pipeline. 3–6 min on a fresh machine. The plugin's "Show pipeline install instructions" command also gives you this exact command with a copy button.

  3. 3. Read a note aloud

    Open any note. Ribbon icon (audio-lines) or command palette → Read note aloud with captions. Space toggles play/pause. Escape closes the HUD.

    First read warms up the model (~30 s). Subsequent reads of unchanged notes replay from cache instantly.
